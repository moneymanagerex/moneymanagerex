일반 보고서 관리자(GRM) 도구 또는 주 메뉴 도구 → 일반 보고서 관리자…를 사용하여 호출합니다.

새 보고서 추가

2가지 옵션을 사용할 수 있습니다.

처음부터 보고서 만들기

  1. 탐색 트리를 사용하여 보고서 이름을 마우스 오른쪽 버튼으로 클릭하세요.
  2. 다음 메뉴 항목을 선택하세요: 빈 보고서 만들기….
  3. 새 보고서 그룹의 이름을 입력합니다.
  4. 보고서 이름을 바꿀 필요가 있는 경우, 마우스 오른쪽 버튼을 클릭하여 보고서 이름 바꾸기… 메뉴 항목을 선택합니다.
  5. 출력, SQL, Lua, 서식 및 설명의 5개 탭이 표시됩니다.
  6. SQL 탭을 선택하고 SQL 스크립트를 편집기 창에 삽입하고 SQL 스크립트가 세미콜론으로 끝나는지 확인한 다음 저장 버튼을 누르세요.
  7. Test 버튼을 눌러 SQL 스크립트를 실행하세요.
  8. 테스트가 통과되면 서식 만들기 버튼을 사용할 수 있게 됩니다. 보고서용 HTML 템플릿을 생성하려면 이를 누르세요.
  9. 실행 버튼을 눌러 출력 창에 HTML 보고서를 가져옵니다.

보고서 매개변수

MMEX 버전 1.3.4의 경우 날짜를 보고서의 SQL 스크립트로 전송할 수 있습니다. &single_date, &begin_date, &end_date의 세 가지 옵션을 사용할 수 있습니다. SQL에 위 매개변수 중 하나가 포함된 경우 달력 위젯의 날짜로 대체됩니다. 예를 들어, 다음 SQL 스크립트는 지정된 날짜의 모든 계정 잔액을 반환합니다. 

with b as  (
    select ACCOUNTID, STATUS
        , (case when TRANSCODE = 'Deposit' then TRANSAMOUNT else -TRANSAMOUNT end) as TRANSAMOUNT
        , TRANSDATE
    from CHECKINGACCOUNT_V1    
    where STATUS <> 'V'
        and TRANSDATE <= '&single_date'
    union all
    select TOACCOUNTID, STATUS
        , TOTRANSAMOUNT ,TRANSDATE
    from CHECKINGACCOUNT_V1
    where TRANSCODE = 'Transfer'
        and STATUS <> 'V'
        and TRANSDATE <= '&single_date'
    )
select a.ACCOUNTNAME, a.ACCOUNTTYPE,
total(TRANSAMOUNT) + a.INITIALBAL as Balance
from ACCOUNTLIST_V1 as a
left join b on a.ACCOUNTID=b.ACCOUNTID
where a.STATUS = 'Open'  and a.ACCOUNTTYPE !='Investment'
group by a.ACCOUNTNAME order by a.ACCOUNTNAME asc;
