Algemene rapport beheerder

Algemene rapportagebeheerder openen

Roep Algemene Report Manager (GRM) aan of gebruik het hoofdmenu Tools → Algemene Report Manager… .

Een nieuw rapport toevoegen

Er zijn 2 opties beschikbaar.

Het rapport vanaf nul maken

  1. Gebruik van de navigatieboom, klik met de rechtermuisknop op het Rapporten label.
  2. Kies het menu-item: Nieuw Leeg Rapport.
  3. Voer de naam in voor de nieuwe rapportgroep en de nieuwe naam.
  4. Hernoem het rapporten indien nodig, door met de rechtermuisknop op het menu-item: Hernoem Rapport te klikken.
  5. Drie tabbladen worden weergegeven: Output, SQL, Lua, sjabloon en Beschrijving.
  6. Kies het tabblad: SQL, voeg het SQL script in het bewerkingsvenster in, Controleer of het SQL script klaar is met een puntkomma, klik vervolgens op de Opslaan knop.
  7. Druk op de knop Test om het SQL script uit te voeren.
  8. Als de test is geslaagd voor Maak Sjabloon wordt knop beschikbaar. Druk erop om een HTML- -sjabloon voor het rapport te genereren.
  9. Druk op de Run knop om het HTML rapport in het uitvoervenster te krijgen.

Rapportparameters

Voor MMEX versie 1.4.0 is er een mogelijkheid om datums over te dragen naar de SQL van het rapport script. Er zijn vier opties beschikbaar: &single_date, &begin_date, &end_date, &budget_years. In het geval dat een SQL een van de bovenstaande parameter bevat, zal het een datum van een hele kalender widget vervangen. Dit SQL-script retourneert bijvoorbeeld alle rekeningsaldi voor de gespecificeerde datum: 

with b as  (
    select ACCOUNTID, STATUS
        , (case when TRANSCODE = 'Deposit' then TRANSAMOUNT else -TRANSAMOUNT end) as TRANSAMOUNT
        , TRANSDATE
    from CHECKINGACCOUNT
    union all
    select TOACCOUNTID, STATUS, TOTRANSAMOUNT ,TRANSDATE
    from CHECKINGACCOUNT
    where TRANSCODE = 'Transfer'
    )
select a.ACCOUNTNAME,
total(TRANSAMOUNT) + a.INITIALBAL as Balance
from ACCOUNTLIST as a, b
where a.STATUS = 'Open'
    and b.ACCOUNTID = a.ACCOUNTID
    and b.STATUS <> 'V'
    and b.TRANSDATE <= '&single_date'
group by a.ACCOUNTNAME order by a.ACCOUNTNAME asc;
